Properties & Events

This document contains the API reference for the Grid Component.

Selector:

Name Description
This is a required input that should be provided in order to create a grid component. Each array item represents a column's configuration. Everytime a new array is assigned to this input a new grid request will be sent.
This is a required input which should contain a POST endpoint which accepts a grid request and return a grid response as specified in the introduction page.
This is an optional input which will make the grid items selectable. By default it's false which means selection is disabled to end users.
This is an optional input which will allow the user to reorder and also show/hide optional columns if enabled. By default it's false which means these column options are disabled to end users.
This is an optional input (object) which can be used to send additional data to the POST endpoint. Everytime this object is changed a new request will be sent automatically which also makes it ideal to programmatically re-initialise the grid by just assigning an empty object if necessary.
By default the grid system is client side paginated. However if your backend supports server-side pagination in accordance with this document, set this value to true.
This is an optional input which will give the grid an aesthetic upgrade for unique UI preferences but does not offer any additional functionality. (It makes the data grid slightly transparent.)
The responseEmit returns the data when it is received from the backend. It contains the records, total number of pages and also an other object (which can be used to send additional data.)
The selectionEmit outputs the selected records to the parent component. This is only useful if the selection input given is true.
This returns the filter objects created by the grid system when the user adds filters.
This returns the button click along with the record which the click occurred so that any operations can be performed on the parent component.
This output will only happen if the columnControl is set to true. This returns the headings array which was modified by the user. This is useful to save the user's column preference for a return visit.
